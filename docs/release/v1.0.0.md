# iqdb-ivf v1.0.0 — Stable

**The inverted-file index is committed.** v1.0.0 delivers the complete IVF
index — k-means training, IVF-Flat and IVF-PQ search, metadata-filtered
search, retrain, and probe tuning — and freezes its public surface under the
SemVer 1.x guarantee: no breaking changes until 2.0. What you build against
today keeps compiling.

## What is iqdb-ivf?

The inverted-file (IVF) approximate-nearest-neighbour layer of the iQDB vector
database. It partitions the vector space with deterministic k-means and answers
each query by scanning only the `n_probes` clusters nearest the query — the
complement to a graph index, more memory-efficient at very large scale and with
more predictable latency, because every query probes the same fixed number of
clusters. Two variants share one surface: **IVF-Flat** (exact intra-cluster
scan) and **IVF-PQ** (Product-Quantization codes scored by ADC, with optional
exact refine). It implements the `iqdb-index` `Index` / `IndexCore` traits, so
it is a drop-in alternative to `iqdb-flat` and `iqdb-hnsw`.

## The 1.0 surface

```rust
use std::sync::Arc;

use iqdb_index::{Index, IndexCore};
use iqdb_ivf::{IvfConfig, IvfIndex};
use iqdb_types::{DistanceMetric, SearchParams, VectorId};

# fn main() -> iqdb_types::Result<()> {
let cfg = IvfConfig::default()
    .with_n_clusters(2)
    .with_n_probes(2)
    .with_training_sample_size(64)
    .with_seed(7);
let mut idx = IvfIndex::new(2, DistanceMetric::Euclidean, cfg)?;

let sample: Vec<Vec<f32>> = vec![
    vec![0.0, 0.0], vec![0.2, -0.1], vec![-0.1, 0.1],
    vec![10.0, 10.0], vec![10.2, 9.8], vec![9.9, 10.1],
];
let refs: Vec<&[f32]> = sample.iter().map(|v| v.as_slice()).collect();
idx.train(&refs)?;

idx.insert(VectorId::from(1u64), Arc::<[f32]>::from(&[0.0, 0.0][..]), None)?;
idx.insert(VectorId::from(2u64), Arc::<[f32]>::from(&[10.0, 10.0][..]), None)?;

let hits = idx.search(&[0.0, 0.0], &SearchParams::new(1, DistanceMetric::Euclidean))?;
assert_eq!(hits[0].id, VectorId::U64(1));
# Ok(())
# }
```

The committed surface (frozen, recorded in `dev/ROADMAP.md`):

- **`IvfIndex`** — `new` (via `Index`), `new_unconfigured`, `train`, `retrain`,
  `dim`, `metric`, `len`, `is_empty`, `is_trained`, `n_probes` /
  `set_n_probes`, `suggest_n_probes`, `pq_refine_factor` /
  `set_pq_refine_factor`, `cluster_stats`, plus the full `IndexCore`
  implementation (`insert`, `insert_batch`, `delete`, `search`,
  `search_batch`, `flush`, `stats`).
- **`IvfConfig`** — the typed configuration with `with_*` builders and
  `validate`.
- **`IvfClusterStats`** — the cluster-occupancy diagnostic.
- **`VERSION`** and the **`Hit`** re-export.

## Highlights

- **IVF-Flat and IVF-PQ on one surface.** Flip `use_pq` to trade a little
  accuracy for a large drop in memory and scan cost; `pq_refine_factor` tunes
  the exact-rerank shortlist, even at runtime.
- **Deterministic.** Same seed + same data → byte-identical centroids and
  search results on every platform.
- **Filter-first search.** Metadata predicates are evaluated before distance;
  the unfiltered hot path carries no per-entry filter branch.
- **Predictable latency.** `n_probes` is the recall/latency dial, with a
  cluster-size-driven `suggest_n_probes` to set it. IVF never widens the probe
  set behind your back.
- **Retrainable.** Rebuild centroids and codebooks over the live corpus when
  `cluster_stats` shows imbalance, without losing data.

## Why this is 1.0

Every Definition-of-Done criterion (`dev/DIRECTIVES.md` §7) is met:

- **Clean everywhere** — `fmt`, `clippy -D warnings` (default and all
  features), `test --all-features`, and `cargo doc -D warnings` pass on the CI
  matrix (Linux, macOS, Windows), on stable and the 1.87 MSRV.
- **Secure** — `cargo audit` + `cargo deny` clean; **zero `unsafe`**
  (`#![forbid(unsafe_code)]`); no `unwrap` / `expect` / `todo!` / `dbg!` in
  shipping code; every allocation bounded; no method panics on hostile input
  (non-finite coverage, wrong dimensions, over-deep filters, and `k == 0` all
  return a clean `Result`).
- **Correct** — IVF-Flat with full probe is checked **bit-for-bit** against the
  exact `iqdb-flat` oracle, by example and by `proptest` property tests; recall
  for both IVF-Flat and IVF-PQ is validated at scale (2 000 vectors, 16-D)
  against the same oracle.
- **Documented** — every public item carries a runnable rustdoc example; the
  README leads with the Tier-1 surface; `docs/API.md` covers the whole API; six
  runnable examples ship in `examples/`.
- **Benchmarked** — `criterion` benches cover the hot paths: IVF-Flat search at
  several probe counts, IVF-PQ search with refine, filtered search, and
  k-means training.

`loom` is **not applicable**: `IvfIndex` has no shared-state or lock-free path —
it is single-writer-internal (`&mut self` for mutation, `&self` for search), so
there is nothing for a concurrency model checker to explore.

## Breaking changes

**None.** This is the first stable tag; the surface is now committed under the
1.x guarantee.

## What's next (post-1.0, additive)

- Residual-PQ (per-cluster codebooks) remains a tracked option; it would be an
  additive config path, not a break.
- Continued validation against `iqdb-eval` as the spine matures.
